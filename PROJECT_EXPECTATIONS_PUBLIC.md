# Project Expectations — Public (What students should deliver)

This file sets clear expectations for the student consulting team and supervisors.

## Format & deliverables

Students should deliver:

1. **Metadata schema** — a practical schema that includes required fields (author, role, creation date, consent status, version, change history, accessibility flags). Include a short mapping to a recognised standard (e.g., Dublin Core or PROV-O).  
2. **Record lifecycle workflows** — 2–3 workflows (Create → Update → Review → Export) that show where metadata is created or updated and how provenance is preserved.  
3. **AI feature specs** — for 3 accessibility features (e.g., plain-language summarisation, Easy-Read generator, captioning). For each feature provide: purpose, inputs, required metadata, transparency requirements and human-review points. Use `AI_FEATURE_EVALUATION_TEMPLATE.md`.  
4. **Ethics checklist & metrics** — a short evaluation checklist and measurable indicators (readability score, metadata completeness, accessibility coverage, authorship visibility).  
5. **Implementation roadmap** — short/medium/long term steps with governance and technology considerations and options for low-connectivity contexts.  
6. **Final report** — 8–12 pages (concise) plus appendices with schema and workflows. Include a one-page executive summary.

## Quality expectations

- Use credible sources (standards bodies, peer-reviewed papers, advocacy organisations).  
- Document authorship for all contributions (Author, Date, Role).  
- Test the schema and workflows using the fictional example records in `/example_records/`. Do not use any private or real case data.  
- Provide a short demo of how an AI feature would work in practice, emphasising the human-review step.

## Assessment indicators (what “good” looks like)

- Practicality: schema usable by non-technical staff.  
- Ethics: explicit controls for consent and prohibited AI uses.  
- Accessibility: clear plan for accessible outputs and human review.  
- Sustainability: vendor-agnostic options and low-connectivity paths.
